Problem Context & Challenges

The organization required a standardized cloud foundation to support multiple workloads, but existing environments were fragmented, manually configured, and lacked centralized security controls.

  • • Inconsistent environment configurations
  • • Limited network segmentation
  • • No centralized security enforcement
  • • High operational overhead due to manual deployments

Before vs After Architecture

Before

  • • Flat network design
  • • Public exposure of services
  • • No centralized security control
  • • Manual deployments

After

  • • Hub-spoke architecture
  • • Centralized firewall
  • • Private endpoints
  • • Fully automated via Terraform

Architecture Decisions

  • Hub-and-Spoke Network: Centralized control point for traffic inspection and segmentation.
  • Azure Firewall: Selected for centralized ingress/egress filtering and logging.
  • Private Endpoints: Eliminated public exposure of services.
  • Terraform Modules: Enabled reusable, environment-agnostic deployments.

Trade-offs & Considerations

  • • Increased initial complexity vs long-term scalability
  • • Higher upfront cost (firewall) vs improved security posture
  • • Strict network controls vs developer flexibility

Architecture Overview

The architecture enforces a Zero Trust model where all traffic flows through a centralized firewall. Workloads are segmented across spoke networks, ensuring isolation and control.

Security Model

  • • Least privilege access across resources
  • • Network segmentation using NSGs
  • • Private connectivity via Private Endpoints
  • • Centralized inspection via Azure Firewall

Implementation Approach

Infrastructure was deployed using modular Terraform design, separating concerns into network, security, and compute layers.

  • • Modular Terraform structure
  • • Environment separation (dev / prod)
  • • Reusable infrastructure components

Key Learnings

  • • Strong architecture upfront reduces long-term complexity
  • • Security must be embedded, not added later
  • • Automation is essential for consistency at scale
